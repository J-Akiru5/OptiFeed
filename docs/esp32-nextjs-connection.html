What is the “App Key”?

The app key is the DEVICE_TOKEN — a static bearer token that the ESP32 sends in the X-Device-Token HTTP header on every request. The backend looks it up in the EnergyDevice table to authenticate the device.

Default Token

#define DEVICE_TOKEN  "esp32-tok-cict-001"

Defined in firmware-main.ino:42. Stored in the database via prisma/seed.ts:46. You can change it — just keep both sides in sync.

Manual Setup Steps

1
Set WiFi credentials firmware-main.ino 
#define WIFI_SSID  "YOUR_WIFI_SSID"
#define WIFI_PASS  "YOUR_WIFI_PASSWORD"
2
Set the server URL firmware-main.ino

Point the firmware at your deployed Next.js backend:

#define INGEST_URL     "https://your-domain.com/api/ingest"
#define FEED_POLL_URL  "https://your-domain.com/api/feed-command?device_id=A4:CF:12:7E:3B:09"
#define SCHEDULE_POLL  "https://your-domain.com/api/schedule-sync?device_id=A4:CF:12:7E:3B:09"
3
Set (or keep) the Device Token & MAC firmware-main.ino Prisma seed 
#define DEVICE_TOKEN  "esp32-tok-cict-001"
#define DEVICE_MAC    "A4:CF:12:7E:3B:09"

The defaults match the Prisma seed. If you change the token here, also update it in prisma/seed.ts and re-run the seed: 

npx prisma db push
npx prisma db seed
4
Flash the ESP32 Arduino IDE

Open firmware-main.ino in the Arduino IDE, select the correct board (ESP32-WROOM-32U) and COM port, then hit Upload.

How Authentication Works

Firmware side (ESP32)

Every HTTP request attaches the token as a header: 

https.addHeader("X-Device-Token", DEVICE_TOKEN);

Used in 5 functions: pollDashboardFeedCommand(), pollScheduleCommand(), postScheduleAck(), postFeedEvent(), sendTelemetry().

Backend side (Next.js)

Each API route reads the header and looks up the device: 

const token = request.headers.get("x-device-token");
const device = await prisma.energyDevice.findUnique({
  where: { token },
});
if (!device) return Response.json("Unauthorized", { status: 401 });

Routes: POST /api/ingest, GET /api/feed-command, GET /api/schedule-sync.

Verification (curl)

Test the connection after flashing:

1. Heartbeat

curl -X POST https://your-domain.com/api/ingest \
  -H "Content-Type: application/json" \
  -H "X-Device-Token: esp32-tok-cict-001" \
  -d '{"device_id":"A4:CF:12:7E:3B:09","event_type":"heartbeat","timestamp":"2026-07-12T12:00:00Z","rtc_ok":true,"feeder_active":false}'

# Expected: 204 No Content

2. Feed Command Poll

curl https://your-domain.com/api/feed-command?device_id=A4:CF:12:7E:3B:09 \
  -H "X-Device-Token: esp32-tok-cict-001"

# Expected: {"feed_requested":false,"feed_request_id":null,"grams":0,"grams_per_feeding":150}

3. Wrong token (should 401)

curl -X POST https://your-domain.com/api/ingest \
  -H "Content-Type: application/json" \
  -H "X-Device-Token: wrong-token" \
  -d '{"device_id":"A4:CF:12:7E:3B:09","event_type":"heartbeat"}'

# Expected: 401 Unauthorized

Troubleshooting

401 Unauthorized

The DEVICE_TOKEN in the firmware doesn’t match any row in the EnergyDevice table. Check both files are in sync:

  • firmware-main.ino → #define DEVICE_TOKEN
  • prisma/seed.ts → energyDevice.create({ token: ... })

Device not found (wrong MAC)

The device_id query parameter (DEVICE_MAC) must match the mac field on the EnergyDevice row linked to the token.

No response / timeout

The SERVER_URL (or INGEST_URL / FEED_POLL_URL) is unreachable. Verify the ESP32 has internet access and the domain is correct.
